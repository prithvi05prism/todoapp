Request Response Lifecycle and HTTP

HTTP is a connectionless protocol where the connection is broken and re-established when the response is ready between the client and the server. It also deals with the connection status freshly every single time it re-establishes it i.e. information regarding each other persists only until the connection exists and for that connection only.
A typical HTTP message consists of a start line, headers and body. A request message differs from a response message. Both computers must speak the HTTP protocol for it to work. The connection between the user and the server is established through the internet (through multiple wired and wireless connections). When a client searches for a website, the browser builds an HTTP request addressed to that URL, this request is then passed on to the client's internet service provider who deliver it to the server which decodes the message and prepares a response HTTP message.
HTTP protocol was initially created for the transfer of HTML files over the internet and became the foundational protocol widely used over the World Wide Web. But this protocol is outdated because anyone can read these requests which might lead to malicious actors taking advantage. Hence HTTPS was introduced which uses a public-private key concept for encrypting the request and response.
